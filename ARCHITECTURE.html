What Claude checks & validates before building
① Right floorplan
Picks Worklist vs List Report vs Object Page based on the user's task. Asks you to confirm before continuing. Wrong floorplan = wasted work.
② Every component exists
Each component is checked against 37 verified SAP schemas. If a component isn't in the registry — the pipeline stops. No hallucinated components ever reach Figma.
③ Correct component for the job
Applies SAP composition rules — which components can go inside which. A Button can live in a Toolbar; it cannot go inside a ShellBar. These rules are enforced automatically.
④ Only real SAP properties
Only non-default properties are set, using documented API values. No invented prop names, no hardcoded colours, no pixel values — only what SAP's API allows.
⑤ ShellBar is always first
SAP Fiori mandates the Shell Bar on every screen. Claude enforces this structurally — it is always the first element in the hierarchy, no exceptions.
✓ validationStatus: pass
The spec is marked pass only when all checks above are green. The plugin refuses to build any spec without this flag. One gate. No exceptions.
🏆 The Winning Insight
SAP library components already know their own height, colours, typography, and internal structure. The plugin's only job is to stack them.

The spec's DynamicPage → slots → content is SAPUI5 code architecture — how components wire in JavaScript. Figma doesn't care about that. Figma is a vertical stack of self-contained SAP instances at screen width. flatWalk() strips the SAPUI5 noise and extracts the visual order.

The Import API Discovery — 3 Attempts

✗
Attempt 1 — figma.importComponentByKeyAsync(key) → HTTP 404
Used component SET keys from search_design_system MCP with this API. Every call returned 404. Reason: importComponentByKeyAsync expects a single variant component key. Our keys are component SET keys (parent container of all variants). Wrong function for the key type.
✗
Attempt 2 — figma.teamLibrary.importComponentSetByKeyAsync(key) → "not a function"
Tried the team library namespace based on docs. The function does not exist in Figma's plugin runtime. figma.teamLibrary only has variable collection methods — no component import.
✓
Attempt 3 — figma.importComponentSetByKeyAsync(key) → WORKS
Correct API: importComponentSetByKeyAsync directly on figma (not figma.teamLibrary). The componentKey values from search_design_system MCP are exactly the right keys for this function. Returns ComponentSetNode → .defaultVariant → .createInstance() → place on canvas.

Key insight: search_design_system MCP returns component SET keys. importComponentSetByKeyAsync accepts component SET keys. Perfect match.

The Import Model — Code

// Keys from search_design_system MCP — these are component SET keys
const SAP_KEYS = {
  ShellBar: '169cfd74c0be329c56b4c79b9404c978ff10cb60',
  Button:   '91805fa199b1fd247d76a9c08bbe0982b49065c4',
  // ... 55 more
};

// Import cache — each key imported once, reused for all instances
const _cache = {};
async function importKey(key) {
  if (_cache[key]) return _cache[key];
  const compSet = await figma.importComponentSetByKeyAsync(key); // ← THE WORKING API
  _cache[key] = compSet.defaultVariant;
  return _cache[key];
}

// Create instance with variant properties
async function sapInstance(specName, variantProps) {
  const key = SAP_KEYS[KEY_MAP[specName]];
  const comp = await importKey(key);
  const inst = comp.createInstance();
  inst.setProperties(variantProps); // e.g. {'Form Factor':'Compact','Type':'Primary'}
  return inst;
}

// flatWalk — strips SAPUI5 slot structure, extracts visual order
function flatWalk(nodes) {
  const result = [];
  function walk(node) {
    if (LAYOUT_CONTAINERS.has(node.component)) {
      // Transparent: DynamicPage, FlexBox, Page, etc — recurse into slots
      Object.values(node.slots || {}).forEach(v => walk(v));
      (node.children || []).forEach(walk);
    } else if (TRANSPARENT_SAP.has(node.component)) {
      // Table etc: extract headerToolbar first, then the Table itself
      if (node.slots?.headerToolbar) walk(node.slots.headerToolbar);
      result.push(node);
    } else {
      result.push(node); // leaf SAP component
    }
  }
  nodes.forEach(walk);
  return result;
}

// Screen builder — flat stack
async function buildScreen(spec) {
  const screen = makeFrame(spec.screen.name, { layout: 'VERTICAL', w: 1440 });
  const nodes = flatWalk(spec.hierarchy);
  for (const node of nodes) {
    const inst = await buildSapNode(node, density, width);
    screen.appendChild(inst);
    inst.layoutSizingHorizontal = 'FILL'; // fill 1440px
  }
}

⚠ Troubleshooting
Import failed errors → SAP Web UI Kit not connected as team library. Go to Assets (Shift+I) → Libraries → enable SAP Web UI Kit.
Components are 1150px wide → Select them and set Resizing to Fill container in the right panel.
Pink/large DynamicPageHeader → Known issue with setProperties for Collapsed variant. Select and manually collapse in Figma.
Syntax error on run → Figma cached old plugin. Remove and re-import from manifest (Plugins → Development → Manage plugins).
